This website is compiled via Jekyll and hosted on Github. To publish a Quickie you need a Github account and a Fork of the Quickie Repo.

Clone your fork to your machine.

{% highlight bash %}~/Code/Jekyll $ git clone git@github.com:mustardamus/quickie.github.com.git quickie-publish Cloning into quickie-publish... remote: Counting objects: 58, done. remote: Compressing objects: 100% (52/52), done. remote: Total 58 (delta 17), reused 0 (delta 0) Receiving objects: 100% (58/58), 11.90 KiB, done. Resolving deltas: 100% (17/17), done. ~/Code/Jekyll $ cd quickie-publish/ ~/Code/Jekyll/quickie-publish $ ls _config.yml _layouts _posts _site css index.html js robots.txt sitemap.xml ~/Code/Jekyll/quickie-publish ${% endhighlight %}

Then create your Quickie in the _posts directory. Use the yyyy-mm-dd-quickie-template.html Template for a starting point.

Commit your changes and push them to your fork.

{% highlight bash %}~/Code/Jekyll/quickie-publish $ git add . ~/Code/Jekyll/quickie-publish $ git commit -m "made article changes" [master 671b9c8] made article changes 2 files changed, 4 insertions(+), 2 deletions(-) ~/Code/Jekyll/quickie-publish $ git push Counting objects: 9, done. Delta compression using up to 2 threads. Compressing objects: 100% (5/5), done. Writing objects: 100% (5/5), 614 bytes, done. Total 5 (delta 3), reused 0 (delta 0) To git@github.com:mustardamus/quickie.github.com.git 46a2455..671b9c8 master -> master ~/Code/Jekyll/quickie-publish ${% endhighlight %}

If you are happy with your changes send in a Pull Request. Done. I'll review your Post and publish it within' some days.
